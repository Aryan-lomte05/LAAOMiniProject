\section{Project Description}
% ============================================================

\subsection{Background and Motivation}

Matrix decompositions --- particularly the Singular Value Decomposition (SVD), QR
factorization, and eigenvalue decomposition --- are the foundational building blocks
of modern data science, machine learning, and scientific computing. They underpin
algorithms for Principal Component Analysis (PCA), dimensionality reduction, latent
semantic analysis, recommendation engines, image compression, and numerical solutions
to partial differential equations.

However, all classical algorithms for these decompositions share a fundamental
bottleneck: the SVD of an $m \times n$ matrix requires $\mathcal{O}(mn\min(m,n))$
floating-point operations. For matrices arising in genomics (millions of genes
$\times$ thousands of samples), text mining (vocabulary size $\times$ document count),
and image databases, this cost is simply prohibitive. Three structural challenges of
modern data motivate a new approach:

\begin{enumerate}[leftmargin=2em, itemsep=3pt]
  \item \textbf{Computational cost}: Classical algorithms are not well-adapted to modern
    parallel architectures such as GPUs and distributed clusters, where memory bandwidth
    is the bottleneck, not raw FLOPS.
  \item \textbf{Memory access}: For matrices too large to fit in RAM (out-of-core), classical
    methods require $\mathcal{O}(k)$ passes over the data. Even for modest $k$, this can
    mean hours of disk I/O.
  \item \textbf{Data imprecision}: Classical SVD computes machine-precision results, which
    is wasteful when the input data is inherently noisy or only approximately known.
    Trading some accuracy for speed is both mathematically sound and practically desirable.
\end{enumerate}

\subsection{The Randomized Solution}

The key insight of Halko, Martinsson, and Tropp (2009) is powerful in its simplicity:
for matrices with \emph{numerical low rank} --- i.e., matrices whose singular values
decay rapidly to zero --- a \emph{random} projection of the matrix captures nearly
all of its important structure. Concretely:

\begin{itemize}[leftmargin=1.8em, itemsep=2pt]
  \item Draw a random test matrix $\Omega \in \mathbb{R}^{n \times l}$ (Gaussian or structured).
  \item Compute the sample matrix $Y = A\Omega \in \mathbb{R}^{m \times l}$.
  \item The columns of $Y$ are random samples from the range (column space) of $A$.
  \item Orthonormalize $Y$ to get $Q$. Then $A \approx QQ^\top A$ with high probability.
\end{itemize}

This is not mere heuristic intuition --- the paper provides \emph{rigorous probabilistic
error bounds} (Theorems 1.1 and 1.2) showing the approximation is optimal up to a small
factor with probability exponentially close to 1. The method has since been adopted in
production libraries: \texttt{sklearn.utils.extmath.randomized\_svd},
\texttt{torch.linalg.svd\_lowrank}, and Facebook's FBPCA library.

\subsection{Our Implementation Scope}

This project implements the complete randomized framework from the paper, covering:
\begin{itemize}[leftmargin=1.8em, itemsep=2pt]
  \item All four Stage A algorithms (Algorithms 4.1, 4.2, 4.3/4.4, 4.5)
  \item Stage B direct SVD (Algorithm 5.1)
  \item Interpolative Decomposition (Section 5.2)
  \item CUR Decomposition (Section 2.1.4)
  \item Empirical verification of Theorems 1.1 and 1.2
  \item Five comprehensive numerical experiments with visualizations
\end{itemize}

\section{Concept Used}
% ============================================================

\subsection{Primary Mathematical Concepts}

\begin{enumerate}[leftmargin=1.8em, itemsep=4pt]
  \item \textbf{Randomized Singular Value Decomposition (Randomized SVD)}\\
    The core algorithm. Replaces the $\mathcal{O}(mn^2)$ classical SVD with a two-stage
    randomized procedure costing $\mathcal{O}(mnk)$ or $\mathcal{O}(mn\log k)$.

  \item \textbf{QR Factorization}\\
    Every $m \times n$ matrix $A$ (with $m \geq n$) can be written as $A = QR$ where
    $Q \in \mathbb{R}^{m \times n}$ has orthonormal columns and $R \in \mathbb{R}^{n \times n}$
    is upper triangular. Used in every stage of the algorithm to orthonormalize sample matrices.

  \item \textbf{Power / Subspace Iteration}\\
    To handle matrices with slowly-decaying singular spectra, we apply the range finder
    to $(AA^\top)^q A$ instead of $A$. Since $\sigma_j((AA^\top)^q A) = \sigma_j(A)^{2q+1}$,
    the effective spectrum decays exponentially faster. Just $q=1$ or $q=2$ is sufficient in practice.

  \item \textbf{Subsampled Randomized Fourier Transform (SRFT)}\\
    A structured random matrix of the form $\Omega = \sqrt{n/l}\,R\,F\,D$ where $D$ applies
    random $\pm 1$ sign flips, $F$ is the DFT matrix applied via FFT, and $R$ subsamples
    $l$ rows. Reduces Stage A cost from $\mathcal{O}(mnl)$ to $\mathcal{O}(mn\log l)$.

  \item \textbf{Interpolative Decomposition (ID)}\\
    Represents $A \approx A_{:,J}\,X$ using $k$ \emph{actual columns} of $A$. The factor
    $X$ satisfies $|X_{ij}| \leq 2$ (bounded entries) and $X_{J,:} = I_k$ (columns
    of $A_{:,J}$ are reproduced exactly). More interpretable than SVD.

  \item \textbf{CUR Decomposition}\\
    $A \approx C\,U\,R$ where $C = A_{:,J}$ is $k$ columns, $R = A_{I,:}$ is $k$ rows,
    and $U = W^\dagger$ (pseudoinverse of the $k \times k$ intersection submatrix
    $W = A_{I,J}$). Entirely in the original data space.

  \item \textbf{Johnson--Lindenstrauss Lemma}\\
    A random linear map $f: \mathbb{R}^n \to \mathbb{R}^l$ approximately preserves
    pairwise distances when $l = \mathcal{O}(\varepsilon^{-2}\log n)$. This underpins
    why random test matrices capture the action of $A$.

  \item \textbf{Measure Concentration (Gordon's Theorem)}\\
    For a Gaussian random matrix $\Omega$, the approximation error $\|A - QQ^\top A\|$
    concentrates sharply around its expectation. The actual error is almost surely very
    close to the average case, making the expected-error bounds practically meaningful.

  \item \textbf{Principal Component Analysis (PCA)}\\
    An application. Given a centered data matrix $X$, PCA computes the $k$ directions
    of maximum variance via the leading $k$ right singular vectors. Randomized SVD
    computes these at $3$--$11\times$ lower cost with negligible accuracy loss.
\end{enumerate}

\subsection{Key Mathematical Framework}

The two-stage decomposition framework:
\[
\underbrace{A}_{m \times n}
\;\approx\;
\underbrace{Q}_{m \times l}\;
\underbrace{Q^\top A}_{l \times n},
\qquad Q^\top Q = I_l, \quad l = k + p,
\]
where $k$ is the target rank and $p \geq 0$ is the oversampling parameter.

\textbf{Theorem 1.1} (Basic error bound): With Gaussian $\Omega$ and $p \geq 2$:
\[
\mathbb{E}\,\|A - QQ^\top A\|_2
\;\leq\; \left(1 + \sqrt{\frac{k}{p-1}}\right)\sigma_{k+1}
\;+\; \frac{e\sqrt{k+p}}{p}\sqrt{\sum_{j>k}\sigma_j^2}.
\]

\textbf{Theorem 1.2} (Power iteration bound): With $q$ power iteration steps:
\[
\mathbb{E}\,\|A - U_k\Sigma_k V_k^\top\|_2
\;\leq\;
\left[1 + 4\sqrt{\frac{2\min(m,n)}{k+1}}\right]^{\!\frac{1}{2q+1}} \sigma_{k+1}.
\]
As $q$ increases, the bracket converges to $1^{1/(2q+1)} \to 1$ exponentially.

\section{Application Area}
% ============================================================

\begin{enumerate}[leftmargin=1.8em, itemsep=5pt]
  \item \textbf{Image Compression}\\
    A grayscale image is an $m \times n$ matrix. A rank-$k$ randomized SVD approximation
    stores only $k(m+n)$ floats instead of $mn$, achieving compression ratio $k(m+n)/(mn)$.
    At $k=50$ on a $256 \times 256$ image, this is 39\% storage with $<9\%$ reconstruction error.

  \item \textbf{Dimensionality Reduction / PCA}\\
    Given a high-dimensional data matrix $X \in \mathbb{R}^{d \times N}$ (e.g., $d=50000$
    gene expressions for $N=500$ patients), classical PCA costs $\mathcal{O}(d^2 N)$.
    Randomized PCA for the top $k=50$ components reduces this to $\mathcal{O}(dNk)$,
    enabling analysis previously considered computationally infeasible.

  \item \textbf{Recommendation Systems}\\
    User--item interaction matrices (Netflix: $10^5$ users $\times$ $10^4$ movies) are
    low-rank in practice. Randomized SVD finds the latent factors faster, enabling
    real-time retraining as new ratings arrive.

  \item \textbf{Large-Scale Numerical Linear Algebra}\\
    For matrices stored on disk or arriving as a stream, randomized algorithms need only
    $2q+1$ passes over the data (vs.\ $\mathcal{O}(k)$ for Krylov methods), making them
    the only feasible option for truly massive datasets.
\end{enumerate}

\section{Tentative Algorithm / Methodology}
% ============================================================

\subsection{Stage A: Randomized Range Finder (Algorithm 4.4 --- Stabilized)}

\textbf{Input}: $A \in \mathbb{R}^{m \times n}$, target rank $k$, oversampling $p$,
power iteration count $q$.

\begin{enumerate}[leftmargin=2em, itemsep=3pt]
  \item Set $l = k + p$ (total sample count).
  \item Draw random Gaussian test matrix:
    $\Omega \in \mathbb{R}^{n \times l}$, $\Omega_{ij} \sim \mathcal{N}(0,1)$ i.i.d.
  \item Compute initial sample matrix: $Y = A\Omega \in \mathbb{R}^{m \times l}$.
  \item \textbf{Power iteration} ($q$ steps, with QR stabilization at each step):
  \begin{enumerate}[label=(\alph*), leftmargin=2em, itemsep=2pt]
    \item Compute QR of $Y$: $(Q_\text{temp}, \_) = \mathrm{QR}(Y)$
    \item Form $Z = A^\top Q_\text{temp}$ and compute QR: $(\tilde{Q}, \_) = \mathrm{QR}(Z)$
    \item Update: $Y = A\tilde{Q}$
  \end{enumerate}
  \item Final orthonormalization: $(Q, R) = \mathrm{QR}(Y)$. Return $Q \in \mathbb{R}^{m \times l}$.
\end{enumerate}

\textbf{Why QR stabilization matters}: Without intermediate QR steps,
floating-point errors in repeated multiplication by $A$ and $A^\top$ cause numerical
collapse --- information about small singular modes is lost. The QR steps cost
$\mathcal{O}(l^2 m)$ extra (negligible) but preserve full precision.

\noindent\textbf{Python implementation}:
\begin{verbatim}
def randomized_range_finder_power(A, l, q=2, random_state=None):
    rng = np.random.default_rng(random_state)
    Omega = rng.standard_normal((A.shape[1], l))
    Y = A @ Omega
    for _ in range(q):
        Q_temp, _ = qr(Y, mode='economic')       # Stabilize
        Z = A.T @ Q_temp
        Q_temp2, _ = qr(Z, mode='economic')      # Stabilize
        Y = A @ Q_temp2
    Q, _ = qr(Y, mode='economic')
    return Q
\end{verbatim}

\subsection{Stage A: Algorithm 4.1 --- Basic Range Finder}

When $q=0$ (fast spectrum or large oversampling sufficient):
\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Draw $\Omega \in \mathbb{R}^{n \times l}$ with i.i.d.\ $\mathcal{N}(0,1)$ entries.
  \item $Y = A\Omega$.
  \item $(Q, \_) = \mathrm{QR}(Y)$. Return $Q$.
\end{enumerate}

\noindent\textbf{Cost}: $\mathcal{O}(mnl + ml^2)$. When $l \ll n$, dominated by matrix multiply.

\subsection{Stage A: Algorithm 4.2 --- Adaptive Range Finder}

For the fixed-precision problem (numerical rank unknown):
\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Initialize: draw $r=10$ Gaussian vectors; form $Y = A\Omega \in \mathbb{R}^{m \times r}$; $Q = []$.
  \item \textbf{While} $\max_i \|(I-QQ^\top)y_i\| > 10\sqrt{2/\pi}\,\varepsilon$:
  \begin{enumerate}[label=(\alph*), leftmargin=2em, itemsep=1pt]
    \item Select the column $y_j$ with largest residual norm.
    \item Orthogonalize against current $Q$: $q_\text{new} = (I-QQ^\top)y_j / \|(I-QQ^\top)y_j\|$.
    \item Append $q_\text{new}$ to $Q$.
    \item Draw a new Gaussian sample, project out current basis.
  \end{enumerate}
  \item Return $Q$.
\end{enumerate}

\subsection{Stage A: Algorithm 4.5 --- SRFT Fast Range Finder}

For large dense matrices where $\mathcal{O}(mn\log l)$ cost is needed:
\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Draw $d_1,\ldots,d_n \sim \mathrm{Uniform}(\{-1,+1\})$; form $AD$ by scaling columns.
  \item Apply FFT column-wise: $\hat{A} = \mathcal{F}(AD) / \sqrt{n}$.
  \item Sample $l$ column indices $I$ uniformly without replacement from $\{1,\ldots,n\}$.
  \item $Y = \sqrt{n/l}\cdot\mathrm{Re}(\hat{A}_{:,I})$.
  \item $(Q, \_) = \mathrm{QR}(Y)$. Return $Q$.
\end{enumerate}

\noindent\textbf{Python implementation}:
\begin{verbatim}
def srft_range_finder(A, l):
    m, n = A.shape
    rng = np.random.default_rng(42)
    signs = rng.choice([-1.0, 1.0], size=n)
    AD = A * signs[np.newaxis, :]              # Apply D
    AF_complex = fft(AD, axis=1) / np.sqrt(n) # Apply F via FFT
    idx = rng.choice(n, size=l, replace=False) # Subsample R
    Y = np.sqrt(n / l) * np.real(AF_complex[:, idx])
    Q, _ = qr(Y, mode='economic')
    return Q
\end{verbatim}

\subsection{Stage B: Algorithm 5.1 --- Direct SVD}

Given orthonormal $Q$ from Stage A:
\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Form $B = Q^\top A \in \mathbb{R}^{l \times n}$.
  \item Compute thin SVD: $B = \hat{U}\,\Sigma\,V^\top$.
  \item Lift back to full space: $U = Q\hat{U}$.
  \item Return truncated factors: $U_k, \Sigma_k, V_k^\top$.
\end{enumerate}

\subsection{Interpolative Decomposition (Section 5.2)}

Represents $A$ using actual columns of $A$:
\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Run Stage A to get $Q$.
  \item Form projected matrix: $B = Q^\top A \in \mathbb{R}^{l \times n}$.
  \item Apply \textbf{pivoted QR} to $B$: $B \cdot P = \hat{Q}\hat{R}$; take first $k$ pivot
    indices $J$.
  \item Selected columns: $C = A_{:,J}$.
  \item Solve least-squares: $\min_X \|A - CX\|_F$. Return $J, X, C$.
\end{enumerate}

\subsection{CUR Decomposition (Section 2.1.4)}

\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Compute column sampling probabilities:
    $p_j^\text{col} = \|A_{:,j}\|_2^2 / \|A\|_F^2$ (leverage scores).
  \item Sample $k$ columns: $\mathrm{col\_idx} \sim \mathrm{Categorical}(p^\text{col})$.
  \item Compute row sampling probabilities:
    $p_i^\text{row} = \|A_{i,:}\|_2^2 / \|A\|_F^2$.
  \item Sample $k$ rows: $\mathrm{row\_idx} \sim \mathrm{Categorical}(p^\text{row})$.
  \item $C = A_{:,\mathrm{col\_idx}}$,\quad $R = A_{\mathrm{row\_idx},:}$.
  \item $U = (A_{\mathrm{row\_idx},\mathrm{col\_idx}})^\dagger$. Return $C, U, R$.
\end{enumerate}

During development, two correctness issues were identified and fixed:

\begin{enumerate}[leftmargin=1.8em, itemsep=4pt]
  \item \textbf{Adaptive Range Finder --- Pool Memory Leak}\\
    \textit{Fix}: Added a pool size cap of \texttt{MAX\_POOL = r + max\_rank}. bounded memory to $\mathcal{O}(m \cdot \text{max\_rank})$.

  \item \textbf{SRFT Range Finder --- FFT Column Selection Order}\\
    \textit{Fix}: Column selection (\texttt{AF\_complex[:, idx]}) is now performed on the
    full complex FFT output, and \texttt{np.real()} is applied \emph{after} selection.
\end{enumerate}
